### API

```
POST   /trigger              → Create Trigger
GET    /trigger/pending       → Oldest pending + recent_activity
POST   /trigger/{id}/ack     → Mark acked
```

### Design Decisions
- Database: shared `data/activity.db` (triggers table alongside events)
- ID format: `trg_<base36-ts><hex-random>` (time-sortable)
- `type`: free string (not enum) — `procrastination`, `sleep`, `study`, `focus`, ...
- `priority`: 0=highest, 1=normal, 2=low
- `status`: `pending` / `acked` only (no `processing`, `error`, `retry`)
- `GET /trigger/pending` returns `{"trigger": null, "recent_activity": [...]}` when empty
- Layering: Router → Service → Repository → SQLite (strict)

### Prompt Format (Final)

```
🔔自动触发|{YYYY-MM-DD HH:MM} {优先级}

{事实描述}

请自行决定是否回复。
```

Priority marks: `Ⅰ` (0=highest), `Ⅱ` (1=normal), `Ⅲ` (2=low).

### Design Principle
Backend stores triggers as JSON. `buildPrompt()` is a pure function on Android. Backend never generates prompt text.
